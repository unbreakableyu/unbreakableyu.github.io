CS50-Lab3-Sort

发表于 2022-11-23 分类于 CS50x 2022

题目要求

Clickhere

概述：想办法猜出Sort1 Sort2 Sort3分别对应哪种算法。

学习收获

算法复杂度

  • Worst-case senario 对应上限，即 upper bound
  • Best-case senario 对应下限，即 lower bound
  • Θ: when upper bound is same as lower bound

Algorithms

Linear Search

最质朴的依次Search。

Worst-case senario: O(n)

Best-case senario: Ω(1)

Binary Search

divide:(sorted array)

  • Sort the array
  • Repeat until the (sub)array is of size 0:
    • middle point
    • if target is at the middle point, stop
    • otherwise, if target is less than what’s at the middle, repeat and change end point to be the left of the middle
    • otherwise, if target is more than what’s at the middle, repeat and change start point to be the right of the middle
      Worst-case senario: O(log n)

      Best-case senario: Ω(1)

Bubble Sort

Compare the adjacent pair and swap. Repeat until the swap-counter is 0.

Worst-case senario: O(n^2)

Best-case senario: Ω(n)

Selection Sort

Find the smallest and add it to the end of the sorted list. Repeat until no unsorted elements remain

Worst-case senario: O(n^2)

Best-case senario: Ω(n^2)

Merge Sort

  • Sort the left half (n > 1)
  • Sort the right half (n > 1)
  • Merge two halves
    Worst-case senario: O(n log n)

    Best-case senario: Ω(n log n)

总结

Name Upper Bound Lower Bound
Selection Sort O(n^2) Ω(n^2)
Bubble Sort O(n^2) Ω(n)
Merge Sort O(n log n) Ω(n log n)
Linear Search O(n) Ω(1)
Binary Search O(log n) Ω(1)

解题思路

题目给出了几组数，分别是随机的(random)，倒序的(reversed)，排序好了的(sorted)。

通过不同算法的运作机制及算法复杂度可以知道，对于排序好了的数，Bubble Sort 一定是最快的；对于随机的数，Merge Sort 最可能是最快的。因此可以首先通过对sorted.txt排序来确定 __Bubble Sort__，然后通过对random.txt排序来确定 __Merge Sort__，最后剩下的自然而然就是 __Selection Sort__。

小岛拜访

在看算法复杂度的时候有一点不太理解Binary Search的算法复杂度是log n而不是log 2^n，然后在这一篇里找到了解释：算法的时间复杂度分析之O(logn)、O(nlogn)
